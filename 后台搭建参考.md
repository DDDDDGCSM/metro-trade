# 后台搭建参考（供其他项目复用）

本文档描述 Trueque Digital（bookforMX）的**统计后台 + 埋点 + 持久化**实现，方便另一个项目参考或复用。

---

## 一、整体架构

| 组件 | 说明 |
|------|------|
| **后端** | Flask（Python） |
| **部署** | Vercel（Serverless） |
| **持久化** | Neon PostgreSQL（免费），未配置时自动回退到内存 |
| **后台入口** | `/admin/stats?token=你的Token`，Token 校验后展示统计 |

---

## 二、核心能力清单

1. **埋点上报**：前端通过 `POST /api/track` 上报事件（PV、分享、点击、提交等）。
2. **持久化**：优先写 Neon Postgres，失败则写内存（适合 Vercel 无盘环境）。
3. **统计后台**：PV/UV、分享/提交/WhatsApp 点击、书籍浏览次数、最近提交明细（含 IP、书名、是否有图等）。
4. **鉴权**：后台用 URL 参数 `token` 校验，Token 建议放在环境变量 `ADMIN_TOKEN`。

## 四、后端实现要点（可直接参考 app.py）

### 4.1 数据库与存储策略

- **环境变量**：任选其一即可  
  `DATABASE_URL` / `POSTGRES_URL` / `NEON_DATABASE_URL` / `SUPABASE_DATABASE_URL`
- **连接**：使用 `psycopg2`，URL 若为 `postgres://` 需改为 `postgresql://`。
- **表结构**：在 `_init_database_if_available()` 中执行：
  - `book_exchange_events`：`id`, `event_type`, `book_id`, `anon_id`, `extra` (JSONB), `ip`, `user_agent`, `created_at`
  - 索引：`event_type`, `created_at`, `book_id`
- **回退**：若未配置 DB 或连接失败，用内存 `list` + `threading.Lock`，并限制条数（如 10000 条）防止 OOM。

### 4.2 核心函数（均在 app.py）

- **`add_event(event_type, book_id=None, anon_id=None, extra=None, ip='', user_agent='')`**  
  写入一条事件，优先 DB，否则内存。
- **`get_events(event_type=None, limit=None)`**  
  按类型、条数查询事件列表（时间倒序）。
- **`count_events(event_type)`**  
  某类型事件总数。
- **`get_distinct_ips(event_type)`**  
  某类型事件的去重 IP（用于 UV）。
- **`get_daily_stats(days=30)`**  
  按天的 PV/UV（基于 `page_view`，按 `DATE(created_at)` 聚合）。

### 4.3 埋点接口

- **路由**：`POST /api/track`
- **请求体**：JSON  
  `event_type`（必填）, `book_id`, `anon_id`, `extra`
- **服务端**：从 `X-Forwarded-For` 或 `request.remote_addr` 取 IP，与 `User-Agent` 一并传给 `add_event()`。

### 4.4 后台页

- **路由**：`GET /admin/stats`
- **鉴权**：`request.args.get('token')` 与 `os.environ.get('ADMIN_TOKEN', '默认Token')` 比较，不对则返回 403 + 简单登录页（表单 GET 提交到 `/admin/stats`，带 `token`）。
- **数据**：  
  - `count_events('page_view')` → PV  
  - `len(get_distinct_ips('page_view'))` → UV  
  - 同理统计 `share`、`exchange_request`、`whatsapp_click`、`book_view`  
  - `get_daily_stats(30)` → 近 30 天 PV/UV  
  - `get_events('exchange_request', limit=50)` → 最近提交，解析 `extra` 得到书名、故事摘要、是否带图等，IP 直接展示（内部用）。

## 五、前端埋点约定（供对方项目对接）

- **上报地址**：`POST /api/track`，Content-Type: application/json。
- **建议事件类型**：`page_view` | `share` | `exchange_request` | `whatsapp_click` | `book_view` 等，与后台统计一致即可。
- **可靠上报**：点击后即将跳转或关闭页面前，用 `navigator.sendBeacon` 或 `fetch(..., { keepalive: true })` 先发埋点，再执行 `window.open` 等，避免请求被取消。

   - 后台：`/admin/stats`，用 URL 参数 `token` 鉴权，展示 PV/UV、行为统计、最近提交明细。
   - 前端：POST /api/track，优先 `sendBeacon`/keepalive，再跳转。

## 七、与本项目差异时的适配建议

- **表名/事件类型**：若对方项目用不同表名或事件名，只需在对方代码里改表名与 `event_type` 字符串即可，结构可保持一致。
- **业务明细**：`exchange_request` 的 `extra` 在本项目中包含 `story_snippet`、`has_image` 等，对方可按业务自定义 `extra` 结构，后台解析时相应修改。
- **Token**：强烈建议对方使用自己的 `ADMIN_TOKEN` 并写入环境变量，不要沿用本项目的默认值。
